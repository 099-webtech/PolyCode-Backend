# Contributing to Quantum

Thank you for helping grow the Quantum language and this learning platform.

## Ways to contribute

- **Lessons** — Improve or add content under `data/beginner`, `data/intermediate`, and `data/advanced`
- **Examples** — Add `.sa` samples under topic folders or `examples/`
- **Projects** — Submit learner projects in `projects/`
- **Quizzes** — Add questions in `quizzes/` (format TBD)
- **Website** — Frontend integration in `website/`
- **Docs** — Toolchain and reference docs in `docs/`

## Content guidelines

1. Every topic folder should keep the standard layout:
   - `README.md` — lesson overview
   - `example1.sa`, `example2.sa` — runnable samples
   - `practice.md` — exercises
   - `notes.md` — supplementary notes
2. All examples must run with `qrun` and/or `quantum` where applicable.
3. Use clear, beginner-friendly language in beginner tracks; assume prior topics in higher levels.
4. Prefer mixed-syntax examples where they illustrate Quantum’s unique value.

## Pull request process

1. Fork the repository
2. Create a branch: `feature/topic-name` or `fix/description`
3. Keep commits focused and messages clear
4. Open a PR with:
   - What changed
   - Which track/level (beginner / intermediate / advanced)
   - How you tested examples

## Code of conduct

Be respectful and constructive. This project is for learners of all backgrounds.

## Questions

Open an issue with the label `question` or contact the maintainers listed in the root `README.md`.
